# Anti-Slop 가드 (한국어) — Layer A

> 모든 초안에 **항상 적용**. 보이스 프로필보다 우선하는 하드 가드.
> 출처: `hardikpandya/stop-slop`(14.0k★, 2026-07-18 확인) 방법론을 한국어로 적응. 원본 CHANGELOG 2026-01-13 추가분(연기된 진심·설명 대신 선언·절대어·강조부사·리듬 패턴) 반영 완료.
> 영어 전용 규칙(em-dash·Wh-시작·부사 전멸·능동태 강제·단문 연속 금지)은 **의도적으로 제외** — 작성자님 보이스를 죽이기 때문(4절 참조).

## 1. 하드 금지 (zero tolerance)

**이항대비·대구** — 텔레그래프된 반전. 기본적으로 "Y다"로 직진한다.
- "X가 아니라 Y다", "문제는 X가 아니다, Y다", "정보가 아니라 연결이다", "단순히 X가 아니라 Y"
- **예외: 글 전체의 논지가 곧 그 대비일 때 한 번만 허용.** 글이 "X 기준을 버리고 Y 기준으로 갈아탄 이야기"라면 그 대비가 주장 자체다. 이때는 논지 문장 1회에만 쓰고, 나머지는 Y만 말한다.
- 판별: 그 문장을 빼면 글의 주장이 사라지는가? 사라지면 논지(허용), 남으면 장식(금지).
- **2회 이상은 무조건 금지.** 같은 대비를 반복하면 주장이 설명으로 주저앉는다.

**부정 나열** — "X도 아니고, Y도 아니고, Z다" 식 빌드업. → Z만 말한다.

**수사적 셋업** — 통찰을 예고만 하고 안 줌.
- "생각해보면", "이게 무슨 말이냐면", "한번 생각해보세요", "그래도 괜찮다"
- → 바로 본론.

**거짓 행위주체** — 무생물에 사람 동사. AI가 행위자를 숨길 때 씀.
- "데이터가 말해준다", "시장이 보상한다", "문화가 바뀐다", "대화가 흘러간다"
- → 사람을 명시. "팀이 그 주에 고쳤다". 안 되면 "당신이".

**관망자 시점** — 장면 위에서 강의.
- "사람들은 보통 ~", "이건 ~때문이다", "~하기 마련이다"
- → 독자를 현장에. "당신은 ~"

**공허한 단언** — 구체 없이 중요성만 선언.
- "이유는 구조적이다", "시사하는 바가 크다", "본질적인 문제다"
- → 그 구체적인 것을 이름으로.

**메타코멘트·요약 오프너** — 글이 제 구조를 announce.
- "이 글에서는", "정리하자면", "결론적으로", "다시 말해", "한 가지만 짚으면"
- → 삭제. 글이 그냥 움직이게.

**긴 서두(throat-clearing)** — 본론 전 헛기침 (brand_dna 금기와 동일).
- "사실 말하자면", "오늘은 ~에 대해 이야기해보려 합니다"
- → 첫 줄부터 훅/본론.

**경구·pull-quote** — 인용하기 좋은 멋부린 한 줄이면 다시 쓴다.

**절대어** — 구체 대신 쓰는 가짜 권위.
- "모두", "누구나", "항상", "절대", "아무도", "다들 ~한다"
- → 숫자·범위로. "다들 이렇게 합니다" → "제가 만난 3곳은 전부".

**연기된 진심** — 없는 친밀감을 만드는 강조.
- "진짜입니다", "정말이에요", "믿기 어렵겠지만", "스며듭니다"
- → 삭제. 사실만 두면 독자가 알아서 믿는다.

**설명 대신 선언** — 어렵다·중요하다고 말만 하고 안 보여줌.
- "이거 진짜 어렵습니다", "이게 진짜 ~라는 겁니다", "이게 실제로 중요한 부분입니다"
- → 어려웠던 장면을 쓴다. "3주 동안 답이 없었습니다".

**안심 완충(hedging)** — 단언해놓고 바로 물러서기.
- "항상 그런 건 아니지만", "완벽하진 않지만", "물론 케바케지만"
- → 삭제하거나 조건을 구체적으로. (진짜 모르는 것을 모른다고 쓰는 건 예외 — 4절)

**한국어 피동·이중피동** — 행위자를 숨김.
- "~되어집니다", "~하게 됩니다", "~인 것으로 보입니다", "결정이 내려졌습니다"
- → 행위자를 앞으로. "제가 정했습니다".

## 2. AI 톤 금기어

- "공유합니다", "정리해봤습니다"(영업톤), "도움이 됐으면 합니다"
- "중요합니다", "핵심입니다"
- 비즈니스 자카온: "한 단계 도약", "게임체인저", "~을 통해 ~할 수 있습니다"(공허)
- 공허한 가능 단언으로 문단 끝맺기: 구체 없는 "~할 수 있습니다"

**강조 목발** — 뜻을 안 더하는 힘주기. 삭제.
- "그게 다입니다", "이게 중요한 이유는", "분명히 말씀드리면", "한번 곱씹어보세요"

**자곤 → 평어 치환**

| 쓰지 말 것 | 대신 |
|---|---|
| 딥다이브 | 뜯어보기, 분석 |
| 인사이트를 얻다 | 알게 되다 |
| 한 발 물러서서 | 다시 보면 |
| 온보딩하다(사람 외) | 붙이다, 도입하다 |
| 러닝포인트 | 배운 것 |
| 리소스가 부족하다 | 사람이/시간이 없다 |

**AI 남용 강조어** — 없어도 뜻이 같으면 뺀다.
- "정말", "진짜", "매우", "굉장히", "단순히", "근본적으로", "결국", "무려"
- 단, "사실"·"솔직히"는 4절 예외 참조.

## 3. 표기 가드

- **사람 수 = "명"**("6명"). 높임은 "분"으로 따로("와주신 분"). 숫자+"분"("6분")은 시간 오해 → 금지.
- **노쇼/저조 프레이밍** — "10명 신청, 6명 참석"은 아쉬움으로 읽힘. 긍정 의도면 "주말 오전인데 6명이나".

## 4. 명시적 비적용 — 이건 금지가 아니다 (작성자님 보이스 보호)

stop-slop 원본은 아래를 금지하지만 **우리는 허용**한다. 이게 작성자님 보이스의 핵심이라서:

- **말줄임표·감정 단편**: "억울하... 아니 아깝다...", "ㅎ;;;;", "미루지 말자!" — 유지.
- **청중 질문 종결**: "관심 있으신 분 계신가요?", "있나요?" — 시그니처. 유지.
- **단문 연속·짧은 hook**: 작성자님의 리듬. 금지 안 함.
- **"사실"·"솔직히"의 자연스러운 사용**: "사실 이걸 노리고 만든 자리였습니다"는 OK. 공허한 메타 오프너로 쓸 때만 금지.

2026-07-18 stop-slop 갱신분 중 아래도 **비적용**:

- **번호·불릿 정보 목록**: 원본은 3항목 나열을 리듬 클리셰로 금지하지만, 작성자님 성공글(팁 정리·용어 정리)의 핵심 포맷이다. 금지 대상은 **수사적 3연 나열**("빠르고, 싸고, 정확합니다")뿐. 정보 목록은 개수 제한 없음.
- **진짜 불확실성 표현**: "아직도 정답을 모르겠지만"처럼 실제로 모르는 것을 밝히는 문장은 hedging 금지에 걸리지 않는다. 단언을 해놓고 되무르는 경우만 금지.
- **질문 후 즉답**: 원본은 금지하지만 작성자님 글의 자문자답 리듬은 유지.
